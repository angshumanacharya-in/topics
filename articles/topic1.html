Execution Lifecycle & Routing Engine

Introduction

In high-scale documentation hubs, traditional full-page reloads introduce severe performance overheads. Our front-end routing layer uses the HTML5 History API alongside non-blocking asynchronous payload fetching. This ensures clean URL states without hash fragments (#) and eliminates state crashes when refreshing pages.

Routing Pipeline

Given a URL path /articles/topic1, the route resolver extracts the slug and dynamically requests the snippet:

fetchPath = appRoot + "/articles/" + slug + ".html?t=" + Date.now()

Algorithm Flow

User Action │ ▼ Extract Slug from Path │ ▼ Fetch Payload with Timestamp │ ▼ Inject Markup into Article Container │ ▼ Update History & Highlight Sidebar

Correctness & Guarantees

Lemma 1: Cache Elimination

By appending ?t=Date.now() to every outbound request, browser cache traps are bypassed, guaranteeing that edits to topic1.html take effect immediately.

Complexity Analysis

  • Route Resolution: Evaluated in O(1) time.
  • DOM Injection: Bounded by payload size O(k).

Overall Resolution Time: O(1)

Implementation Code

async function loadArticle(slug) {
    const fetchPath = `/articles/${slug}.html?t=${Date.now()}`;
    const res = await fetch(fetchPath);
    const html = await res.text();
    document.getElementById('articleContainer').innerHTML = html;
}
import time

def get_cache_bust_path(slug: str) -> str:
    timestamp = int(time.time() * 1000)
    return f"/articles/{slug}.html?t={timestamp}"
