Topic 1: Execution Lifecycle & Routing

Deep Dive into the Front-End Architecture

This chapter provides a technical breakdown of how article requests are resolved, processed, and updated inside the DOM runtime without crashing or introducing race conditions.

1. Route Resolution Algorithm

When a user opens a link or refreshes the page, the client-side router executes a pattern match against window.location.pathname. The extraction process operates as follows:

function resolveInitialRoute() {
  const path = window.location.pathname;
  const match = path.match(/\/articles\/([^\/]+)/);

  if (match && match[1]) {
    loadArticle(match[1], false);
  } else {
    // Fallback to the first available topic
    const firstLink = articleList.querySelector('.nav-link');
    if (firstLink) loadArticle(firstLink.dataset.slug, true);
  }
}

2. Asynchronous Fetch & Cache Management

Static file updates can sometimes be cached aggressively by web browsers. To ensure that editing an HTML file immediately reflects on the live site without hard-refreshing, every fetch request attaches a dynamic timestamp parameter:

const articlePath = `/articles/${slug}.html?t=${Date.now()}`;

This guarantees that the browser always fetches the latest modified version directly from your server while preserving clean visual paths in the address bar.

3. State Synchronization

Whenever an article payload successfully loads into the DOM container, two visual synchronization tasks execute automatically:

  1. Sidebar Link Highlighting: The active class is transferred to the matching data-slug anchor tag.
  2. Bottom Navigation Calculation: The DOM tree calculates neighbor nodes relative to the current position to populate the Previous and Next action buttons.
