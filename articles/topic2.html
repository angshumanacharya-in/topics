Styling, Responsiveness, and Extensions

Now that you understand routing and execution lifecycle, this chapter focuses on modifying and scaling the user interface to suit customized themes and complex article content.

Customizing CSS Variables

The core styling system uses modern CSS custom properties. You can easily modify colors and sizing directly inside style.css without hunting through nested rules:

:root {
  --header-bg: #09090b;      /* Deep black header background */
  --header-text: #ffffff;    /* Header font color */
  --sidebar-bg: #18181b;    /* Sidebar background */
  --sidebar-text: #a1a1aa;  /* Default sidebar text color */
  --active-color: #3b82f6;  /* Highlight accent color */
  --sidebar-width: 280px;   /* Width of sidebar drawer */
}

Scaling Up to 100+ Topics

As you add dozens or hundreds of HTML files into your articles/ directory, keep the following tips in mind for maximum performance and clean organization:

  • Keep file slugs consistent and lower-case (e.g., database-indexing.html).
  • Ensure every new article link in index.html has an accurate data-slug="..." property.
  • Break long topics into digestible subheadings (<h2> and <h3>) to preserve visual clarity.
