Getting Started

Setup and Configuration Guide

Setting up your documentation environment requires zero complex dependencies. Because the core engine runs on vanilla HTML5, CSS3, and JavaScript, you can host these files on any static web server or local development stack.

Prerequisites

Before launching your site, ensure you have the following installed or configured:

  1. A local HTTP development server (such as Node.js Express, Live Server, Python's http.server, or Nginx).
  2. Basic understanding of HTML markup for formatting documentation content.
  3. A modern web browser with JavaScript enabled.

Quick Start Guide

To initialize the documentation structure locally, verify your folder layout matches the structure below:

/my-docs-site
├── index.html
├── style.css
├── app.js
├── logo.png
└── articles/
    ├── introduction.html
    ├── getting-started.html
    ├── topic1.html
    └── topic2.html

Testing the Routing Engine

When running a local HTTP server, open your browser and navigate through the topics. Try the following tests to confirm stability:

  • Click between different sidebar topics and observe that the URL updates cleanly (e.g., /articles/getting-started) without standard hash fragments.
  • Press the F5 key or refresh button on any article URL. The app should resolve the route correctly without throwing a 404 error.
  • Click the logo in the top-left corner to toggle the deep-black sidebar off and on.
